# Grenco Medical — Brand & Product Assets Portal

A clean, professional, self-serve asset portal for **pharmacies, partners, and press**.
One brand (**Grenco Medical**), one product (the **Grenco Medical Elite II**), with search,
a lightbox preview, per-file and "Download all" downloads, a how-to video hub, a Product
Specialist training + certification course, and pharmacy marketing-material requests.

It's a **plain static site** — HTML + CSS + vanilla JS, no build step, no framework. It is
designed to live inside the main Grenco Medical site repo under `brand-portal/`, and is
served at `/brand-portal/`.

> **Compliance note:** The Elite II is a prescription medical device. These assets are for
> approved pharmacy/partner/press use only and must not be used to advertise or promote its
> supply to the public. Direct patients to speak with their prescriber.

---

## 1. Run it locally

Serve the **site root** (so `/brand-portal/` and the shared `../assets/` resolve), then open
the portal:

```bash
# from the main site repo root
python3 -m http.server 8091
# then open http://localhost:8091/brand-portal/
```

---

## 2. Edit the content (the file most people touch)

Open **`assets/data/assets.js`**. It's heavily commented. It defines:

- `PORTAL_CONFIG` — title, tagline, intro, request/order emails, usage note.
- `PORTAL_BRANDS` — the single `grencomedical` brand (colors, fonts, links).
- `PORTAL_CURRENT` — the current product lineup.
- `PORTAL_PRODUCTS` — the Elite II product + its asset folders, plus the logos set.
- `PRODUCT_INFO` — official copy: highlights, full description, spec/regulatory table, box.
- `PORTAL_TRAINING` — the Product Specialist course (learn modules + quiz).

Image paths are relative to `brand-portal/`. The shared site assets live one level up at
`../assets/` (e.g. `../assets/device-marble.jpg`).

**Do not invent facts.** Unknown specs are shown as `To be confirmed`. No pricing is displayed.

---

## 3. Where the real files come from (Dropbox sync)

`assets/data/synced.js` is regenerated by **`scripts/dropbox-sync.mjs`** (run in GitHub
Actions — see `.github/workflows/dropbox-sync.yml`). It lists the Grenco Medical Dropbox
folders, builds a thumbnail for every file, commits the originals + thumbnails under
`assets/synced/`, and writes `window.PORTAL_SYNCED`. When present, the real Dropbox
folders/files replace the seeded in-repo ones.

Sync sources (`PRODUCTS` in `scripts/dropbox-sync.mjs`):

| Portal name | Slug | Notes |
| --- | --- | --- |
| Grenco Medical Elite II | `elite-ii` | commits originals for in-portal download/zip |
| Grenco Medical Elite II — Videos | `elite-ii-videos` | how-to-use + how-to-clean; folded into the Elite II hub |
| Grenco Medical Logos | `grencomedical-logos` | transparent PNG thumbnails; flat "Logos" folder |

**Secrets** (GitHub → Settings → Secrets): `DROPBOX_APP_KEY`, `DROPBOX_APP_SECRET`,
`DROPBOX_REFRESH_TOKEN`. The workflow runs with `working-directory: brand-portal` because
the portal is a subfolder of the main site repo.

## Files

- `index.html` — page shell (nav, hero, sections, lightbox)
- `assets/css/styles.css` — all styling (CSS variables at top to retheme)
- `assets/js/app.js` — all behavior (rendering, search, lightbox, downloads, training)
- `assets/data/assets.js` — **content** (brand + product + files + copy + training)
- `assets/data/synced.js` — Dropbox-synced overlay (regenerated by the Action)
- `scripts/dropbox-sync.mjs` — the Dropbox → `synced.js` sync
- `.github/workflows/dropbox-sync.yml` — scheduled sync workflow
